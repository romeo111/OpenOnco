# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is a Claude Code skill for systematic cancer treatment research. It produces evidence-ranked treatment plans that maximize patient life extension, covering standard of care through experimental therapies. The skill generates both Markdown reports and companion JSON data files.

The project includes an **autoresearch loop** (inspired by Karpathy's autoresearch pattern) that iteratively improves report quality by editing a research strategy file, running benchmark cases, measuring quality, and keeping or discarding changes.

**This is NOT a traditional software project** — it is a prompt-driven skill definition with Python scoring/orchestration utilities.

## File Inventory

### Core Skill
- **`cancer-research-skill.skill`** — The skill definition file (zip archive containing the skill bundle)
- **`SKILL.md`** — Skill manifest with frontmatter (name, description, triggers) and the full workflow specification (input parsing, 6-tier research strategy, rating system, output structure, guidelines)
- **`output_template.md`** — Markdown and JSON templates for report output. Reports have 9 sections: Executive Summary, Standard of Care, Ranked Treatments, Clinical Trials, Emerging Pipeline, Combination Matrix, Supportive Care, Methodology, and Rating Appendix
- **`cancer_research_scorer.py`** — Python CLI tool for validating and re-scoring research output JSON

### AutoResearch Loop
- **`generate_cases.py`** — Input calibration generator. Produces `benchmark_cases.json` with realistic patient cases for a given cancer site/demographic. Includes curated head & neck cases and generic templates for other sites.
- **`benchmark_cases.json`** — Fixed test set (generated by `generate_cases.py`). Do NOT modify between loop iterations.
- **`evaluate.py`** — Quality scoring harness. Takes a report JSON and produces a single 0-100 score across 7 dimensions. Imports from `cancer_research_scorer.py`.
- **`strategy.md`** — Agent-editable research strategy. The ONLY file the loop agent modifies. Contains search templates, budget allocation, rating calibration, inclusion criteria.
- **`program.md`** — Loop instructions for the agent. Defines the edit→run→measure→keep/discard cycle.
- **`run_experiment.py`** — Experiment orchestrator. Runs all benchmark cases, scores each, computes mean, logs to `results.tsv`, returns keep/discard decision.
- **`results.tsv`** — Append-only log of experiment iterations (timestamp, iteration, mean_score, decision, note).
- **`experiment_reports/`** — Directory for per-case prompts and report JSON files.
- **`research_db.py`** — Unified SQLite database (`research.db`) storing ALL research results: treatments, trials, sources, scores, tumor board reviews. Supports ingest, query, and export.
- **`research.db`** — SQLite database file. Tables: cases, reports, treatments, clinical_trials, combinations, supportive_care, sources, evaluations, board_reviews, iterations.

### Virtual Tumor Board
- **`virtual_tumor_board.py`** — 5 virtual oncologists with opposing philosophies who adversarially review reports. Three modes: (1) heuristic scoring (no LLM), (2) LLM debate prompt generation (4-round structured debate), (3) quick review prompt. The doctors are:
  - **Dr. Evidence** (Evidence Purist) — only trusts Phase 3 RCTs, demands HRs and p-values
  - **Dr. Aggressor** (Aggressive Interventionist) — maximum intensity, combinations, surgery
  - **Dr. Guardian** (Patient-Centered Integrationist) — QoL, toxicity, supportive care, palliative
  - **Dr. Precision** (Precision Medicine Advocate) — biomarkers, genomics, targeted therapy
  - **Dr. Frontier** (Clinical Trialist / Innovator) — trials, ADCs, bispecifics, vaccines

## AutoResearch Loop Architecture

```
┌─────────────────────────────────────────────────────┐
│  program.md (loop instructions — agent reads this)  │
│                                                     │
│  while true:                                        │
│    1. Edit strategy.md (one targeted change)        │
│    2. Run run_experiment.py                         │
│    3. Check mean_score vs best_score + 1.0          │
│    4. Keep or discard strategy.md change             │
│    5. Log to results.tsv                            │
└─────────────────────────────────────────────────────┘
         │                        ▲
         ▼                        │
┌─────────────┐         ┌─────────────────┐
│ strategy.md │────────▶│ run_experiment.py│
│ (editable)  │         │ (orchestrator)   │
└─────────────┘         └────────┬────────┘
                                 │
                    ┌────────────┼────────────┐
                    ▼            ▼             ▼
              ┌──────────┐ ┌──────────┐ ┌──────────┐
              │ Case 1   │ │ Case 2   │ │ Case N   │
              │ research │ │ research │ │ research │
              └────┬─────┘ └────┬─────┘ └────┬─────┘
                   │            │             │
                   ▼            ▼             ▼
              ┌──────────────────────────────────┐
              │         evaluate.py              │
              │   7 dimensions → 0-100 score     │
              └──────────────────────────────────┘
```

| Karpathy Pattern | Cancer AutoResearch |
|---|---|
| `train.py` (agent edits) | `strategy.md` (agent edits research strategy) |
| `prepare.py` (fixed eval) | `evaluate.py` (fixed quality scorer, 0-100) |
| `val_bpb` (single metric) | `quality_score` (0-100, higher is better) |
| `program.md` (loop instructions) | `program.md` (loop instructions) |
| `results.tsv` | `results.tsv` |

## Running the Tools

### Generate Benchmark Cases
```bash
python generate_cases.py --site "throat and neck" --age 45 --sex male --count 10
python generate_cases.py --site "lung" --age 60 --sex female --count 8
```

### Evaluate a Report
```bash
python evaluate.py report.json                    # Score with summary
python evaluate.py report.json --verbose           # Detailed breakdown
python evaluate.py report.json --score-only        # Just the number
python evaluate.py report.json --json-output s.json # Save scores as JSON
```

### Run an Experiment
```bash
python run_experiment.py                           # All cases
python run_experiment.py --single-case HN-003      # One case
python run_experiment.py --dry-run                 # Generate prompts only
```

### Virtual Tumor Board
```bash
python virtual_tumor_board.py report.json                          # Heuristic scoring (no LLM)
python virtual_tumor_board.py report.json --heuristic              # Same as above (explicit)
python virtual_tumor_board.py report.json --generate-prompt        # Full 4-round debate prompt
python virtual_tumor_board.py report.json --quick-prompt           # Short single-round review
python virtual_tumor_board.py report.json --case benchmark_cases.json --case-id HN-001  # With patient context
python virtual_tumor_board.py report.json -o debate.json           # Save results
python virtual_tumor_board.py report.json --format prompt -o debate_prompt.md  # Save prompt to file
```

### Research Database
```bash
python research_db.py init                                         # Initialize: ingest cases + all reports
python research_db.py ingest report.json --case-id HN-001          # Ingest single report
python research_db.py ingest-all --iteration 0                     # Ingest all reports from experiment_reports/
python research_db.py query dashboard                              # Full dashboard
python research_db.py query treatments --top 20                    # Top treatments across all cases
python research_db.py query trials                                 # All clinical trials
python research_db.py query scores                                 # Quality scores per case
python research_db.py query sources --type journal                 # Sources by type
python research_db.py query combos                                 # Combination strategies
python research_db.py query supportive                             # Supportive care approaches
python research_db.py query board                                  # Tumor board doctor averages
python research_db.py export treatments --format csv               # CSV export
python research_db.py export full --format json                    # Full JSON export
```

### Validate and Re-score (existing scorer)
```bash
python cancer_research_scorer.py <input.json>
python cancer_research_scorer.py <input.json> --output <output.json>
python cancer_research_scorer.py <input.json> --validate-only
```

No external dependencies — all tools use only Python stdlib (`json`, `argparse`, `datetime`, `re`, `os`, `sys`, `subprocess`, `signal`, `time`, `pathlib`).

## Quality Score Dimensions (evaluate.py)

| Dimension | Max Points | What It Measures |
|---|---|---|
| Structural Integrity | 15 | JSON validates, all sections present, min counts |
| Evidence Depth | 25 | OS/PFS/ORR/HR/sample-size data density |
| Tier Coverage | 10 | All 6 research tiers represented |
| Rating Calibration | 15 | Score spread, range, rationale completeness |
| Source Quality | 15 | URL count, validity, type diversity |
| Clinical Relevance | 10 | Biomarker specificity, side effects, trial status |
| Combo/Supportive Coverage | 10 | Combination strategies + supportive care depth |

## Rating System

Treatments are scored 1–10 using a weighted composite:

| Factor | Weight |
|---|---|
| Evidence Level | 30% |
| Survival Benefit | 30% |
| Accessibility | 15% |
| Safety Profile | 15% |
| Biomarker Match | 10% |

Scores are clamped to [0, 10], composite is rounded to 1 decimal place. The scorer re-sorts treatments descending by composite and reassigns ranks.

## Key Architectural Decisions

- The skill uses **6 tiers of web search** (SoC → Approved Targeted → Clinical Trials → Experimental → Combination Strategies → Supportive Care), performing 10–20 searches per query
- Reports are saved to `/mnt/user-data/outputs/` with filename format `cancer_research_[cancer_type]_[date].{md,json}`
- The JSON schema in `output_template.md` defines the contract that `cancer_research_scorer.py` validates against — keep these in sync
- `evaluate.py` imports `validate_report` and `compute_composite` from `cancer_research_scorer.py` — keep these compatible
- `strategy.md` is the only file the loop agent may edit — all other files are fixed infrastructure
- `results.tsv` is append-only — never delete or modify existing rows
- All output must include a medical disclaimer; this tool is for research/educational purposes only
