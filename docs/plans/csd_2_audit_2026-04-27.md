# CSD-2 Baseline Audit — Drug `ukraine_registration` Coverage (2026-04-27)

Read-only audit of every YAML in `knowledge_base/hosted/content/drugs/`.
Goal: ground the Wave 1 authoring agents in the current state of
`regulatory_status.ukraine_registration` so each agent knows exactly
which fields it must fill, verify, or replace placeholders for.

> **Note on `null` semantics.** `UkraineRegistration.registered` and
> `reimbursed_nszu` default to `False` in the Pydantic schema, so on load
> they are never literally `None`. This audit reads the raw YAML to
> distinguish *explicit* values from default-`False`. Where the YAML key
> is missing entirely it is reported here as the value Pydantic would
> coerce to (`false`) rather than `null`, **except** for
> `reimbursed_nszu`, where 3 files do leave the key absent (rendered as
> blank below).

### Highest-value gaps

- **54 drugs (32%) have no `last_verified` date.** Every Wave 1 author
  edit must set `last_verified: 2026-04-27` (or the actual date the
  agent looked at МОЗ Реєстр / NSZU formulary).
- **39 drugs are flagged `reimbursed_nszu: true` but carry zero
  `reimbursement_indications`** (e.g. `DRUG-ABIRATERONE`,
  `DRUG-ANASTROZOLE`, `DRUG-DEXAMETHASONE`, `DRUG-DOXORUBICIN`,
  `DRUG-ETOPOSIDE`, `DRUG-FLUDARABINE`). NSZU онкопакет coverage is
  always indication-scoped — empty list means "we believe it's covered
  but never wrote down for what". This is the largest single clean-up
  job.
- **3 drugs lack the `reimbursed_nszu` key entirely** (durvalumab,
  interferon-alpha, tremelimumab). Treat as authoring stubs.

## 4. Summary — what is most missing

Ranked by impact on Wave 1 authoring:

1. **`reimbursement_indications` populated**: only 32 / 167 drugs (19%)
   carry any indication list. Of the 71 drugs flagged
   `reimbursed_nszu: true`, **39** still have an empty list. This is
   the highest-ROI cleanup.
2. **`last_verified` dates**: 54 drugs have none. Wave 1 must stamp
   every drug it touches.
3. **`registration_number`**: 9 carry the placeholder; the remaining
   158 are mostly `null` / absent. Filling these is a stretch goal —
   МОЗ Реєстр lookups are slow and not blocking for plan rendering.
4. **Cost orientation** (`cost_uah_reimbursed`, `cost_uah_self_pay`,
   `cost_last_updated`, `cost_source`): out of scope for CSD-2 baseline,
   tracked separately in `docs/plans/ua_ingestion_and_alternatives_2026-04-26.md`.
