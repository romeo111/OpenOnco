# UA full-quality — Phase 2+ plan addendum (2026-05-12)

## Context (post-Phase-0/1 launch)

The site at `openonco.info` is already bilingual: every disease/case page
under `/docs/cases/*` has a mirror at `/docs/ukr/cases/*`. The full
`/ukr/` tree is generated by `scripts/build_site.py` reading the same
KB. So **most** of Phase 2 is implicit: when Phase 1 lands and the site
is rebuilt, the upgraded `*_ua` fields flow into the rendered pages.

Hand-authored UA HTML (verified 2026-05-12): `docs/ukr/about.html`,
`capabilities.html`, `ask.html`, `clinical-gaps.html`, `gallery.html`,
`index.html`, `kb.html`, `specs.html`, `try.html` — already translated,
line counts match EN, register reads cleanly.

## Phase 2 — site verification + UI string sweep (2 sequential steps)

### 2A. Site rebuild + spot-check (1 agent, ~30 min)

**Allowlist (read+write):** `docs/**`, `scripts/build_site.py`,
`scripts/site_*.py`. Do NOT touch `knowledge_base/`.

**Steps:**
1. Pre-flight (worktree branch, clean tree).
2. `C:/Python312/python.exe scripts/build_site.py` (regenerate
   everything). Must exit 0.
3. `git status --short` should now show changes under `docs/`.
4. Sample 18 random files and read end-to-end:
   - 6 from `docs/ukr/cases/` (4 disease auto-cases + 2 BMA cases)
   - 6 from `docs/ukr/disease/` (if exists, else more cases)
   - 3 from `docs/ukr/` root (about, capabilities, ask)
   - 3 from `docs/ukr/kb/` (if exists)
5. For each sampled page:
   - Search for runs of 4+ consecutive English words.
   - Search for `STUB` / `TODO` / `XXX` left in UA.
   - Confirm gene symbols / trial acronyms stay Latin.
   - Confirm drug first-mention pattern `<UA> (<EN INN>)` survives render.
6. Write findings to `docs/reviews/ua-phase2-render-check-<DATE>.md`.
7. Stage rebuilt `docs/` only. Commit. Push branch. Stop.

If sample fails: do NOT mass-edit rendered HTML — diagnose whether the
problem is in the source (KB still has issues, send back to Phase 1
re-run for that entity type) or in the render code (Phase 2B target).

### 2B. Render-layer UA strings (1 agent, sequential — small surface)

**Allowlist:** `knowledge_base/engine/render*.py` and any module the
render layer imports for UI strings (search for `target_lang`).

**Task:** every UI label, section header, "ask your doctor" boilerplate,
glossing prompt, error/empty-state message has a publication-quality
UA branch. Use the same glossary
(`knowledge_base/translation/ua_terminology.yaml`).

**Validation:** rebuild site after edits; diff `/docs/ukr/` byte counts
vs `/docs/`; ratio between 0.92 and 1.20 expected (UA usually slightly
longer). Investigate outliers.

## Phase 3 — spec deltas (parallelizable, 2 agents)

For each of the 16 docs in `specs/uk/`:
1. `git log --since=<last-uk-sync-date> -- specs/<basename>.md` to find
   EN delta commits.
2. Propagate substantive changes into UA spec.
3. Don't re-translate sections that already exist in `specs/uk/`.

**Partition:**
- spec-1: `CHARTER`, `KNOWLEDGE_SCHEMA_SPECIFICATION`,
  `SOURCE_INGESTION_SPEC`, `DATA_STANDARDS`.
- spec-2: the other 12.
